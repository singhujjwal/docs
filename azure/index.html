Account and Subscription

Your company has an account and you get access to the portal.azure.com but if you don't have a subscription, you can't do anything. In that case either you need to be assigned to a subscription or a tenant or you should have an owner role to create a subscription in that account. The account can be alternatively be called as tenant. Here is a link that describe all types of roles in Azure

Directory -> tenant -> Subscriptions (I see one subscription in one directory account is company account)

This defines a role is constructed using json, the similar way AWS IAM policies work.
